    Generating mouse movement with ANN

    Cezar Augusto Cordeiro de Lima
    Oct 15, 2018 · 4 min read

    Many researchers have focused in identifying whether a series of actions was performed by a humans or bots, but there is a lack of articles about how to generate mouse movements that performs like real user actions. In this article, I try to create an Artificial Neural Network (ANN) that generates theses actions.

    Mouse events

    The first step is understanding how mouse events work and what factors may distinguish a robot from a human being.

    Let's use a software that listens for these events as example: your browser. It is simple to program a script that tracks all movements users perform in a web page. These events generated due movement, click, double click, etc., have basic properties like position and timestamp and these characteristics are useful to decide whether an action is human or not.

    The position is a pair of coordinates X and Y that represents where the mouse pointer was located when the event happened, while timestamp is the instant when it was triggered.

    Bots

    When it comes to bots, frequently, they are created to perform repetitive tasks in a fast way. Think about video game bots, players that don't want to keep playing the same tasks again and again tend to use a bot to accomplish them.

    Since game developer are aware of it, they are able to use some strategies to avoid it. One of them is tracking mouse movements to verify if there is a pattern and if it is something that a real player could perform.

    Back to browser example, we can see that if mouse pointer moves from an origin point to a destination, there will be many other movement events between this two and the path traveled by the pointer will not be a straight line with constant time between the actions.

    In order to mitigate these problems some approaches have been proposed. One of them is approximate users movement paths using Bezier Curves, this way instead of a direct line linking origin and destination a curve with several points is generated.

    Although it seems to be a good solution, these are well known curves and games developers could try to focus on methods to identify them.

    The solution proposed here is a neural network to generate movements with human patterns.

    Neural Network

    A simple Artificial Neural Network is proposed to mimic human behavior. It is composed of an input layer with two units, X and Y, this pair corresponds to the destination where the pointer has to be moved to; one hidden layer with 128 units and ReLU activation; and the output layer with 198 values.

    from tensorflow.keras.models import Sequential
    from tensorflow.keras.layers import Dense, Reshape    model = Sequential()
    model.add(Dense(128, activation='relu', input_dim=2))
    model.add(Dense(198, activation='linear'))
    model.add(Reshape((99, 2)))
    model.summary()

    After the output layer is used a Keras Reshape layer, so the network output is composed of 99 points. So the network is designed to have at maximum 99 points between origin and destination.

    Training

    For training data, a dataset was generated with mouse movements and clicks from several users with a total of 10,770 samples.

    The dataset was divided in:

    • 9,770 origin/destination training pairs;
    • 1,000 test pairs.

    Before training the network some pre-processing has been applied to the data. First our origin/destination pairs have been transformed into deltas, then the users paths were moved to begin at origin (0, 0) and scaled to have values between -1 and 1.

    import numpy as np# Loads input dataset like [[xo, yo], [xd, yd]]

    Results

    With the trained network some random sampling was performed, the image below is a comparison between values generated by the network and users.

    It is possible to notice that there is some noise at the end of each path, so the path moves around the destination. To avoid this problem, it is possible to remove all points after we reach destination or when we are very close to it.

    The above image shows a more complex path, generated from small segments. In this case it is easy to see that generated paths need some adjustments, as said before.

    Beyond this noises, another negative thing about the proposed solution is that it is deterministic, that is, for equal pairs (X, Y) the same path will be created. This means that if for a give task always the same points need to be reached, then the path will be the same.

    In order to avoid this problem, future works may focus in generative models to provide data that changes for same input, like Variational Autoencoders (VAE), for example. Future works should also validate if the generated data is labeled as human or bot.

    Important note: I do not recommend using the approaches seen here for gaming bots.
